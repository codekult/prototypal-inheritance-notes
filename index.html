Prototypal Inheritance in JS

Let’s see some examples of JS protoypal inheritance, or for better saying, behavior delegation or OLOO (Objects Linking to Others Objects), that work as a better/cleaner way than using “classes” and “constructors”.

Prototype Delegation

Delegate properties from a delegate prototype to a new or existing object. If you try to access the delegated properties on the new object, it will search through the prototype chain until find them.

Example:

var myNewObj = Object.create(delegateProto);

Concatenative Inheritance

Build an object adding or copying properties from one or multiple exemplar prototypes.

Example:

var myNewObj = Object.assign({}, exemplarProto01, exemplarProto02);

Mixing up both previous aproaches:

var myNewObj = Object.assign(Object.create(delegateProto), exemplarProto01, exemplarProto02);

Notes: Object.create is part of the ES5 spec and is supported on modern browsers. Object.assign is part of the ES6 spec, so in order to use it you have to transpile the code or use a library with some similar utility or polyfill 1.

So simple as that, and you’ve the foundations to start using prototypal inheritance in more complex patterns, without the need to mimic classical inheritance or using constructors.

Alternative methods

Modules

Use the module pattern to encapsulate and reuse code instead of delegate or copying behavior. This a valid approach and if you want to share or “inherit” behavior between modules you can mix this pattern with the previous methods, allowing to pick and choose which features you require.

Examples:

var importedModule = require('importedModule');
// Or with ES6 modules
// import importedModule from './importedModule.js';

var myNewObj = Object.assign({}, importedModule);

Functional Mixins

In this case we’re going to favor composition over inheritance and using functional mixins to achieve it, in a similar way as Object.assign works (let’s say we going to share a interface, instead of a prototype or class).
